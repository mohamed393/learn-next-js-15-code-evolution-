the usage of replace in link component ar next.js
Use replace when you do not want users to return to the previous URL via Back, such as:
login or logout redirects
ex
<Link href="/about" replace>
      About us
    </Link>
replace on the Link component means navigate without
 adding a new entry to the browser history stack.
 ---------------------------------------
layout.tsx vs template.tsx
A good mental model is:
layout = persistent shell
template = fresh wrapper each navigation
So the practical rule is: default to layout,
and only use template when you specifically need a reset/remount behavior
